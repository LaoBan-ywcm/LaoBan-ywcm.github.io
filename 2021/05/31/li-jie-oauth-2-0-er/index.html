实现客户端

在OAuth2.0中获取令牌的方式有很多种，这里采用的授权码类型，客户端将资源拥有者引导至授权服务器，授权服务器通过重定向的方式将授权码返回给客户端，客户端在用授权码换取令牌。还有其他获取令牌的方式，比如隐式许可类型、客户端凭证许可类型、资源拥有者凭据许可类型和断言许可类型

客户端源码：https://github.com/oauthinaction/oauth-in-action-code/blob/master/exercises/ch-3-ex-2/completed/client.js

通过阅读上面代码来学习客户端的处理流程

/callback是处理授权服务器重定向后的接口，在请求令牌时又指定了重定向链接redirect_uri: client.redirect_uris[0]，这是在OAuth 2.0规范中要求的，如果在请求授权码时使用了重定向URI，那么在请求令牌时也必须要带上和请求授权码时相同的URI。

在请求令牌时，要同时指定client_id和client_secret，冒号分割，用Base64编码，通过basic 认证的方式进行传递。Oauth 2.0并没有规定要通过何种方式进行认证，可以根据自己的场景选择。

在代码中还对state值进行了判断if (resState != state)。如果不加这个参数，那么无论是谁请求/callback接口，程序都会尝试用该授权码去获取访问令牌，很有可能会被暴力破解出授权码。为了防止这种情况，OAuth 2.0要求在请求授权码时要带上state参数，授权服务器要原封不动的将state值通过重定向发送给客户端，客户端拿到授权码时要校验state值是否和开始请求授权码的state值相同

/fetch_resource是用来访问受保护资源的接口，访问受保护资源只需要带上令牌就可以了，OAuth 2.0规定了三种携带令牌的方式：

  1. 使用Authorization头部
  2. 使用表单请求
  3. 使用URL编码的查询参数

第二种方式要求受保护资源必须支持表单传递参数，并且API支持POST方法，这样就对受保护资源提出了要求。第三种方式由于将令牌写进了URL，安全性就大大降低，很可能会通过日志暴露出来。通常第一种方式是最安全最灵活的

refreshAccessToken是用来通过刷新令牌更新令牌的，刷新令牌的作用前一篇已经讲过了，如果有疑问的可以翻阅之前的博客。

通过刷新令牌更新令牌时需要设置grant_type: 'refresh_token'。检验令牌是否可用的唯一办法就是去使用他，因为令牌随时都可能被撤回，比如检测过期时间这种方式并不能确定令牌是否可用。

    var inToken = null;

    if (auth && auth.toLowerCase().indexOf('bearer') == 0) &#123;

        // 从表单中获取令牌
    &#125; else if (req.body && req.body.access_token) &#123;

        &#125; else &#123;

        &#125;;

    &#125;);

getAccessToken是用来获取令牌的函数，会通过我们之前说过的三种方式来获取令牌，需要注意的是OAuth 2.0中规定了Bearer是不区分大小写的，但是Bearer空格之后的令牌本身是区分大小写的。

nosql.one是从数据库中查询令牌的方法。一般授权服务器和受保护资源是共享数据库的，但是也有用令牌存储特殊信息的方式，就是JWT，它可以使用受加密保护的JSON携带信息。

在OAuth2.0整个流程中授权服务器第一次接受到的请求就是生成授权页面，在这个过程中是不会传递客户端密码的，只会传递客户端ID，获取到客户端ID后，因为客户端ID是通过浏览器传输的，非常容易就可以拿到。所以必须要与客户端注册时的重定向URI对比，防止冒充客户端。

在这个过程中还要校验客户端想要获取的权限是否超过注册时的权限范围，然后把这次请求中的一些信息存储起来，将key返回

在之前生成授权页面时，返回了一个reqid字段，在前端界面，这个key是不会显示在页面上，提交表单时会自动提交的。如果没有获取到该值，则说明该客户端发这个请求之前并没有请求授权页面，所以是非法请求

生成授权码时需要客户端ID和密码，之前讲过有三种方式可以传递，这里只做了通过header头传递和表单传递，由于header头里传递是做了URL编码的，所以这里需要先解码。

然后就是做了一些校验，ID是否存在，密码是否正确。然后生成令牌（注意返回的token_type字段和scope）和访问令牌。然后通过json格式返回

这里也提供了刷新令牌的操作，和请求令牌只是类型不同，请求令牌时是authorization_code，刷新令牌时是refresh_token
