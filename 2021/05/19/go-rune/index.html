Golang rune类型

在说rune类型之前，先穿插一些编码的基础知识

ASCII、Unicode和UTF-8

ASCII

ASCII码诞生之初是用来转换英语和二进制的，因为在计算机内部，所有的信息都是二进制的。

一个字节(byte)有八位(bit)，可以表示的数字范围00000000 - 11111111，共有2^8中可能性，既256。

ASCII规定了128个字符的编码，比如A字母，对应的ASCII码是0100 0001，十进制是65。!符号，对应的是0010 0001，十进制是33。

ASCII只用到了一个字节的后7位，第一位始终为0

Unicode

ASCII码只规定了英语和一些字符的编码，但是其他语言会有更多的字符，ASCII码就满足不了了，所以出现了Unicode，Unicode规定了世界上所有字符对应的二进制码，可以通过中日韩Unicode编码表查询中文对应的Unicode码

UTF-8

Unicode虽然规定了每个字符对应的二进制，但是并没有规定如何存储这些数据，它只是个对照表。

比如汉字一对应的Unicode十六进制是4E00，二进制是100111000000000，有十五位。龟对应的Unicode十六进制是9F9F，二进制是1001111110011111，有十六位。他们都至少需要两个字节，别的字符还有占三、四个字节的。

这样问题就来了，程序要怎么知道多少个字节表示一个字符？

UTF-8就是用来解决这个问题的，它最大的特定就是它是一种变长的编码方式

UTF-8和Unicode的关系：UTF-8是Unicode的一种实现方式

UTF-8的编码规则其实很简单，就两条：

  1. 对于单字节的符号，第一位是0，后面7位是对应的Unicode码，所以UTF-8是兼容ASCII的
  2. 对于n字节的符号，第一个字节的前n位都设置为1，后面紧跟一位设置为0，后面字节的前两位设置为10。剩下的没有提及的位都用来填充Unicode码

下面是编码规则表

Unicode符号范围     |        UTF-8编码方式
(十六进制)          |       （二进制）
----------------------+---------------------------------------------
0000 0000-0000 007F | 0xxxxxxx
0000 0080-0000 07FF | 110xxxxx 10xxxxxx
0000 0800-0000 FFFF | 1110xxxx 10xxxxxx 10xxxxxx
0001 0000-0010 FFFF | 11110xxx 10xxxxxx 10xxxxxx 10xxxxxx

比如要对一进行UTF-8编码，它的十六进制是4E00，对于规则表第三行，然后将对应的Unicode码100111000000000填充进去，UTF-8编码就是11100100 10111000 10000000，转为十六进制就是E4B880

Little endian 和 Big endian

数据存储的方式有两种大端(Big endian)和小端(Little endian)，比如E4B880，采用大端存储就是E4B880，如果采用小端存储就是80B8E4。

如果是单机环境是没有什么问题的，如果是网络通信，可能传递过来的数据大端方式，而本机采用的是小端方式，这样解析数据就会出现问题。但是一般网络传输都是采用大端方式

一般x86的服务器都是采用小端方式，可以查看操作系统采用的方式

~$ lscpu
Byte Order:          Little Endian

Rune 类型

在Golang中字符串只是一个可读的字节片，从前面的部分知道，有很多字符得需要很多个字节才能表示，所以当我们对字符串进行类似切片中的索引操作时就会出现问题

var a = "这是test"
fmt.Printf("a len: %d\n", len(a))
fmt.Printf("a[:1]: %s\n", a[:1])
fmt.Printf("a[:3]: %s\n", a[:3])
fmt.Printf("% x\n", a)

// a len: 10
// a[:1]: �
// a[:3]: 这
//     e8 bf 99 e6 98 af 74 65 73 74

输出a[:1]时，因为切割字符e8不正确，没有在UTF-8找到对应的字符，所以输出了不可见字符

%x输出每个字节对应的十六进制

但是使用rune类型就可以避免这种问题，先看一下rune的定义

// byte is an alias for uint8 and is equivalent to uint8 in all ways. It is
// used, by convention, to distinguish byte values from 8-bit unsigned
// integer values.
type byte = uint8

// rune is an alias for int32 and is equivalent to int32 in all ways. It is
// used, by convention, to distinguish character values from integer values.
type rune = int32

它底层是用四个字节存储数据，在Unicode标准中有一个定义code point，就是指单个字符对应的Unicode码，比如这对应code point是U+8fd9，对应的UTF-8十六进制就是e8bf99。所以rune类型就是对应Unicode标准中的code point，这样在切割字符串时就不会有问题了

var a = []rune("这是test")
fmt.Printf("a len: %d\n", len(a))
fmt.Printf("a[:1]: %s\n", string(a[:1]))
fmt.Printf("a[:3]: %s\n", string(a[:3]))
fmt.Printf("% x\n", a)

// a len: 6
// a[:1]: 这
// a[:3]: 这是t
// [ 8fd9  662f  74  65  73  74]

对字符串进行for range操作时，每次获得的对象都是rune，所以也可以避免上述问题

var a = "这是test"
for _, v := range a &#123;
    fmt.Printf("% s", string(v))
    // 这是test
&#125;
